#1 Write a Blog on the Difference b/w HTTP 1.1 vs HTTP 2

HTTP

  • HTTP is the short form of Hypertext Transfer Protocol.
  • Developed by Timothy Berners-Lee
  • It is the foundation for World Wide Web and its invention in 1989.
  • It is used by browers in order to load web pages

Example:

  • Client browser sends HTTP request to web server once we enter URL such as http://swasticblog.com
  • The web browser sends an HTTP response to the client with the contents of the webpage.

HTTP 1 Protocol

  • It had official recognition in 1996
  • HTTP methods are Get, Put or Post.
  • It used text-based commands for HTTP requests.
  • Request targets such as URL or Port are used.
  • The new response headers allowed multiple file types.

Example:

  • Plain Text
  • HTML
  • Images
  • Sound
  • Video
  • Other Mulitmedia File

HTTP 1.1 Protocol

  • It was released in 1997.
  • As we can see in HTTP 1 implementation, one major problem was that connection needed to be established ater each request.
  • To solve this problem HTTP 1.1 came up with a keep-alive concept that helped to send multiple requests over a single connection.
  • It can load one request at a time.
  • Hence one request per one TCP - Transmission Control Protocol connection is possible.
  • It added many performance enhancements.

Example:

  • Caching
  • Request Pipelining
  • Head-of-Line Blocking
  • Keepalive Connections
  • Transfer-Encoding
  • Byte-Range Requests

HTTP 2 Protocol

  • It was released in Feb.2015 by Internet Engineering Task Force- (IETE).
  • It was the second major version of the most useful internet protocol,HTTP.
  • It was derived from the earlier experimental (SPDY) - Speedy Protocol.
  • Though HTTP 1.1 was much faster than HTTP 1, it had some problems, most importantly, it was not making use of TCP connection completely.
  • Each connrction was sending one request at a time.
  • This problem was solved in Http 2 and multiple concurrent requests could be sent.
  • It allows servers to "push" responses into client caches proactively.
  • It is fully multiplexed.
  • It is binary and not textual.
  • It uses header compression in order to reduce overhead
  • It interleaves multiple requests/responses in parallel without blocking on anyone.
  • It uses a single TCP connection to deliver multiple requests/responses (in parallel).

STREAMS

  • To achieve this parallel request over a single HTTP connection, HTTP 2 uses the concept of streams.
  • That is, each request being sent from the client has a unique stream id attached behind the scenes.
  • This helps the client and server identify the calling and receiving endpoints.
  • One can think of each stream as an independent channel for communication.
  • It is less error-prone than HTTP/1.
  • It removes unnecessary HTTP/1.x workarounds.

Example:

  • Image Sprites
  • Domain Sharing

"A JavaScript Object is a collection of named Values having state and behavior (properties and method)".

Example:

  • Person
  • Mobile Phone
  • Car
  • Pen
  • Tv
  • Personal Computer etc.

Take the Case of Mobile.

All Mobile has the same properties, but the property values differ from Mobile to Mobile. All Mobile have the same methods, but the methods are performed at different times.

Let's have an example of my mobile phone and list out its properties(Features):

  • ◆ Model: Xiaomi Redmi 10
  • ◆ Version: Android 11, MIUI 13
  • ◆ Color: Caribbean Green
  • ◆ Battery: 6000 mAh

Taking the above as a reference, I'll stress up on objects, Object Properties, and Methods.

1. Objects

Code assign asimple value (Xiaomi Redmi) to a variable named mobile:

let mobile="xiaomi Redmi";

  • Objects are variables too. But objects can contain many values.
The following code assigns many values (Xiaomi Redmi, MIUI 13, Caribbean Green,and so on) to a variable named Mobile:

let mobile={Model:"xiaomi Redmi", Version:"MiUi 13", Color:"Caribbean Green", Weight:"603g", Battery:"6000 mAh", Rating:4.5};

The values are written as name: value pairs (name and value separated by a colon)

Syntax

let <object-name>={key1:"value1", key2:"value2",..... keyN:"valueN"};

  • ◆ JS objects is "JavaScript objects are containers for named values"

2. Object Properties

  • The Name: values pairs (in JavaScript objects) are called properties.

let mobile={Model:"xiaomi Redmi", Version:"MiUi 13", Color:"Caribbean Green", Weight:"603g", Battery:"6000 mAh", Rating:4.5};

  • The object properties can be different primitive values, other objects, and functions.
  • Properties can usually be changed, added, and deleted, but some are read-only.
  • ◆ The syntax for Adding a property to an object is :

ObjectName.ObjectProperty = PropertyValues;

  • ◆ The syntax for Deleting a property from an object is :

delete ObjectName.ObjectProperty;

  • ◆ The syntax to Access a property from an object is :

objectName.property        // Mobile.Model

                                    // or

objectName["property"]   // Mobile["Model"]

                                    // or

objectName[expression]    // x = "Model"; Mobile[x]

JavaScript properties is "Properties are the values associated with a JavaScript object"
